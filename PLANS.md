2. Add architecture and Character Pipeline documentation.
3. Add ADRs for Cocos, rigid node rigs, and JSON animation.
4. Add initial schemas and examples.
5. Add implementation tasks for the first milestone.

### Done when

- A new contributor or Codex session can identify the vision, boundaries, technical baseline, and next task without chat history.
- Schema examples validate against their contracts once the validator is implemented.
- Framework code and project-specific content have explicit boundaries.
