# Red Cap Target Remade fixture

This directory is the canonical engine-neutral source for the supplied
transparent Red Cap Remade art. `source-parts/` preserves the 19 supplied PNG
files byte-for-byte. `parts/` contains deterministic, import-safe RGBA crops
whose filenames are the canonical `partId` values; source files are never
rewritten. `reference/full_character.png` is the intended assembled-proportion
reference.

`source-asset-map.json` is the explicit, fail-closed mapping from supplied
filenames and crop rectangles to canonical Character Pipeline `partId` values.
`source-annotation.json` is authored in the common top-left source canvas.
`rig-layout.json` is generated by the Rig Layout Generator and must not be
edited by hand.

Regenerate the derived PNGs and layout from the repository root:

```bash
pnpm exec node pipelines/character-asset-intake/scripts/prepare-mapped-assets.mjs \
  examples/red-cap-target-remade/source-asset-map.json
pnpm --filter @gameai/rig-layout-generator generate:red-cap
pnpm --filter @gameai/character-asset-intake reconstruct:red-cap
```

The final command reconstructs the neutral pose from `originalRect` centers,
proximal pivots, draw order, and the 326×892 common canvas before any hierarchy
or engine is involved. It writes a side-by-side comparison and fails if the
alpha-silhouette IoU is below `0.8`. Left/right names are always the
character's anatomical sides, so anatomical left appears on the viewer's
right in the front-facing reference.

The Cocos acceptance project contains an AssetDB import mirror at
`assets/gameai/red-cap-target-remade`. Tests require its source documents and
derived PNG bytes to match this canonical fixture. The mirror's
`source-asset-map.json` retains `sourceFile` as provenance while Cocos consumes
only each verified `importFile`.
